# Development of a Gas and Smoke Detection System Using Raspberry Pi

라즈베리파이와 MQ-2 센서를 활용하여 가스 및 연기를 실시간으로 감지하고, 위험 발생 시 능동 부저를 통해 경보를 울리는 IoT 시스템 구축 프로젝트입니다.

> 본 프로젝트는 센서 내부의 화학적 반응 원리와 부저의 물리적 구동 방식을 학습하며,
> 하드웨어의 전압 규격 차이에 따른 분리 회로 구성 및 bz.off()를 통한 안정적인 자원 해제 메커니즘을 심층적으로 다룹니다.\

## 개요

라즈베리파이의 GPIO 인터페이스를 통해 입력 장치(MQ-2)와 출력 장치(Buzzer)의 상호작용 원리를 이해하고, 가스 감지 시스템의 회로 구성 능력을 익힙니다. MQ-2 센서의 산화물 반도체 원리와 능동 부저의 발진 회로 구조를 바탕으로 실시간 경보 로직을 구현하며, 이전 실험에서 학습한 인터럽트 개념을 확장하여 장치 특성에 맞는 안전한 종료 과정을 실증합니다.

## 핵심 특징

- **실시간 위험 감지**: 가스 및 연기 유입 시 즉각적인 "가스 감지됨" 상태 전환 및 부저 경보 수행
- **하드웨어 전압 최적화**: MQ-2(5V)와 능동 부저(3.3V)의 전압 요구사항을 반영한 전원 분리 배선 설계
- **논리 반전 제어**: 가스 감지 시 센서값이 0(LOW)으로 전환되는 디지털 출력 특성을 소프트웨어 조건문에 반영
- **객체 지향 하드웨어 제어**: Buzzer 및 DigitalInputDevice 객체를 활용하여 하드웨어를 소프트웨어와 동기화

## 하드웨어 아키텍처

본 프로젝트는 [라즈베리파이의 BCM 핀 배열](https://www.raspberrypi.com/documentation/computers/raspberry-pi.html#gpio)을 기준으로 다음과 같이 설계되었습니다.
- **MQ-2 가스 센서**: GPIO 17 (VCC: 5V 연결)
- **능동 부저**: GPIO 18 (VCC: 3.3V 연결)

## 회로 구성 이론

- **MQ-2 감지 원리**: 산화주석 반도체가 가스와 반응하여 저항이 감소하는 화학적 변화를 전압 신호로 변
- **능동 부저 동작**: 내장된 발진 회로를 통해 3.3V 전원 공급 시 즉시 일정한 경보음을 출력하는 간편한 제어 방식 활용
- **회로 안전성**: 5V와 3.3V 전압 규격 차이를 준수하여 부품 손상 및 라즈베리파이 보드 고장 위험을 사전에 차단

## 실행 및 설치

### 1. 의존성 설치

본 프로젝트는 라즈베리파이 OS에 내장된 gpiozero 라이브러리를 사용합니다.

### 2. 서버 실행

터미널에서 소스코드가 있는 디렉토리로 이동 후 'python3 main2.py' 또는 'python3 main2-1.py' 명령어를 입력합니다.

### 3. 종료(Interrupt)
- 실행 중 Ctrl + C를 입력하면 KeyboardInterrupt가 발생하며 프로그램이 종료됩니다.
- 종료 시 모든 LED의 value를 0으로 설정하여 잔광 현상을 방지합니다.

## 주요 이슈 해결
- 터미널 실행 환경 차이: 터미널에서 종료 시 일부 LED가 점등 상태로 유지될 수 있으나, 코드 하단의 value = 0 로직을 통해 안정적인 소등을 보장함.
- 하드웨어-소프트웨어 동기화: 코드상의 핀 번호(예: carLedRed = 2, carLedRed = LED(2) )가 단순한 변수가 아닌 실제 물리적 주소임을 인지하고, 결선 오류 발생 시 디버깅 과정을 통해 제어 공학의 기초 원리를 파악함.
- 버전 관리: 시스템 내 파이썬 버전 간 혼선을 방지하기 위해 python3 명령어를 명시적으로 사용하여 라이브러리 구동 안정성을 확보함.
- 프로그램 종료 메커니즘의 한계와 인터럽트(SIGINT) 이해: 임의의 키 종료를 위해 외부 라이브러리 설치를 시도했으나, 별도 설치 없이 운영체제 차원에서 즉각 반응하는 Ctrl + C 인터럽트가 하드웨어 제어 및 안전 종료에 가장 효율적임을 학습함.
